Réalisé avec : ELBASAN Kendal, DECOURT Enzo

Qu'est-ce que PRTG ?

PRTG (Paessler Router Traffic Grapher) est un logiciel de supervision réseau tout-en-un sans agent. Il surveille en temps réel l'état, la disponibilité et les performances d'une infrastructure IT.

  • Détecter les pannes ou anomalies avant impact utilisateur
  • Analyser les performances (CPU, RAM, bande passante, latence…)
  • Prévenir les incidents via alertes (email, push…)
  • Produire graphiques, rapports et tableaux de bord

Fonctionnement de base

PRTG repose sur une hiérarchie :

  • Groupe — catégorie (ex. Réseau, Serveurs)
  • Appareil (Device) — machine/équipement (IP ou nom DNS)
  • Capteur (Sensor) — élément surveillé (Ping, CPU, trafic…)

Protocoles : Ping, SNMP, WMI (Windows), SSH (Linux), HTTP/HTTPS, NetFlow/sFlow…

Installation (Windows)

PRTG s'installe uniquement sur Windows (10/11 ou Server 2016+).

  1. Télécharger la version Free (100 capteurs) sur le site officiel Paessler
  2. Lancer l'installateur .exe, accepter la licence
  3. Saisir email et mot de passe admin, laisser ports par défaut
  4. Patienter 5-10 min puis accéder à https://localhost

Découverte automatique

  1. Clic droit sur "Probe locale" → "Ajouter un groupe de découverte automatique"
  2. Entrer la plage IP (ex. 192.168.1.1 – 192.168.1.254)
  3. Lancer la découverte → PRTG scanne et crée automatiquement appareils + capteurs de base

Capteurs essentiels

Capteur Ping

  1. Clic droit sur appareil → Ajouter capteur → rechercher "Ping"
  2. Sélectionner "Ping standard" → Valider
  3. Vert = OK, Rouge = hors ligne

Trafic ports fibres (switch)

  1. Activer SNMP sur le switch (communauté "public")
  2. Ajouter capteur → "SNMP Traffic" → sélectionner le port fibre
  3. Surveille bande passante entrante / sortante / totale

Alertes par email

  1. Configuration → Notifications → Créer notification "Email"
  2. Configurer SMTP (ex. Gmail : smtp.gmail.com, port 587, TLS)
  3. Créer un déclencheur sur le capteur : Down → Sévérité Haute → Email
  4. Tester l'alerte pour vérifier la réception

Bonnes pratiques

  • Toujours ajouter un capteur Ping par appareil
  • Nommer clairement groupes, appareils et capteurs
  • Limiter les capteurs inutiles (Free : max 100)
  • Coupler PRTG avec GLPI pour tickets automatisés
